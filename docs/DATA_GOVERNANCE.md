# Data Governance

## 데이터 등급

| 등급 | 예시 | 허용 범위 |
|---|---|---|
| Public | 공개 법령, 공개 매뉴얼 | 라이선스와 출처 확인 후 사용 |
| Team | 합성 문서, 팀 내부 실험 결과 | 팀 저장소/승인된 서비스 |
| Confidential | 실제 내부 매뉴얼, 계약/운영 자료 | 명시적 승인, 최소 권한, 외부 전송 검토 |
| Restricted | 개인정보, 인증정보, 영업비밀 | 기본적으로 프로젝트 사용 금지 |

## 데이터 카드 필수 항목

- 데이터셋 이름과 버전
- 소유자와 승인자
- 수집 경로와 확인 날짜
- 이용 약관/라이선스
- 포함 문서 수, 형식, 언어
- 개인정보/민감정보 여부
- 전처리 및 익명화 방법
- 접근 권한과 보존/삭제 날짜
- 알려진 편향과 누락

## 원칙

- 필요한 최소 데이터만 수집합니다.
- 원본, 청크, 임베딩, 캐시, 백업의 삭제 경로를 모두 정의합니다.
- 외부 문서는 URL만 저장한다고 저작권 문제가 사라지는 것이 아니므로 이용 조건을 확인합니다.
- 평가 데이터는 학습/튜닝 데이터와 분리해 누수를 방지합니다.
- 문서 버전과 검색 인덱스 버전을 연결해 답변을 재현할 수 있게 합니다.
