## 추천 순위

### 1. T02 공공입찰 제안 적합성 분석기

- 공개 RFP 확보가 쉽고 내부 문서는 합성 데이터로도 현실적인 데모가 가능합니다.
- 단순 Q&A를 넘어 필수 조건 누락, 보유 역량 매칭, 근거 표 생성까지 확장할 수 있습니다.
- 정량 평가가 쉽습니다: 요구사항 추출 재현율, 근거 정확도, 누락 탐지율.

### 2. T01 연구실 실험 안전 코파일럿

- SOP와 SDS의 충돌/주의사항을 함께 찾는 장면이 명확해 시연 효과가 큽니다.
- 표, PDF, 화학물질명 등 다양한 문서 처리 문제를 경험할 수 있습니다.
- 안전 분야이므로 답변 보류, 경고, 출처 검증 설계를 제대로 보여줄 수 있습니다.

### 3. T03 장애 대응 기억 저장소

- 소프트웨어 팀이 직접 사용자 역할을 할 수 있어 요구사항 검증이 빠릅니다.
- 버전이 다른 공식 문서와 사내 Runbook의 충돌 탐지, 과거 장애 유사도 검색이 차별점입니다.
- 합성 장애 보고서와 오픈소스 문서만으로도 구현 가능합니다.

### 4. T06 지역 축제 운영 의사결정 도우미

- 문서 Q&A에 날씨·교통 같은 최신 외부 정보를 결합하는 확장이 자연스럽습니다.
- 담당자별 체크리스트, 비상 대응 브리핑, 전년도 회고 검색으로 역할별 UX를 만들기 좋습니다.

## 차별화 기능 아이디어

- **근거 충돌 감지**: 내부 매뉴얼과 최신 외부 규정이 다르면 양쪽 근거와 날짜를 함께 표시
- **역할별 답변**: 관리자, 현장 담당자, 신규 직원에게 서로 다른 상세도와 권한 적용
- **변경 영향 분석**: 외부 규정이 갱신되면 영향을 받는 내부 문서와 조항을 자동 제안
- **답변 보류 버튼**: 근거 부족 시 억지 답변 대신 추가로 필요한 문서를 안내
- **문서 신선도 표시**: 버전, 게시일, 마지막 검토일을 답변 신뢰도에 반영
- **평가 근거 뷰어**: 검색된 청크, 재정렬 점수, 최종 인용 포함 여부를 개발자 화면에서 확인

## 선정 기준

다음 질문에서 5개 이상을 명확히 답할 수 있는 주제를 선택합니다.

- 1주 안에 합법적으로 사용할 문서 30개 이상을 확보할 수 있는가?
- 내부 문서를 합성하더라도 실제 업무 흐름을 설득력 있게 재현할 수 있는가?
- 사용자 페르소나와 반복 질문이 구체적인가?
- 답변의 정답 또는 근거 적합성을 사람이 판정할 수 있는가?
- 단순 챗봇과 다른 핵심 기능을 한 문장으로 설명할 수 있는가?
- 잘못 답했을 때의 위험과 답변 보류 기준을 정의할 수 있는가?
- 최종 시연을 5분 안에 문제-해결-효과 순서로 보여줄 수 있는가?
