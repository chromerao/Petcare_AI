# PetCare Expert Guide MVP

## 최우선 작업

최우선은 모델 선택이 아니라 **질문 범위, 신뢰할 출처, 정답 판정 기준을 고정하는 것**입니다. 이 세 가지가 없으면 답변은 그럴듯해도 정확도를 측정할 수 없고, 규정이 바뀌었을 때 어떤 문서를 갱신해야 하는지도 알 수 없습니다.

MVP는 두 축을 함께 다룹니다.

- 보호자 케어: 건강 이상 신호, 독성 섭취, 식이·체중, 행동, 생활 환경, 예방 관리
- 시설 운영: `동물위탁관리업` 중 반려견·반려묘 호텔/유치원의 입퇴실, 위생, 사고, 환불, 규정 확인

모든 답변은 문서 근거 답변과 AI 일반 조언을 구분합니다. 법률 자문, 수의학적 진단, 처방, 약물 용량 지시를 대신하지 않습니다.

## 핵심 사용자 질문

- 신규 입실 시 무엇을 확인하고 어떤 기록을 남겨야 하는가?
- 이상 증상 또는 전염성 질환이 의심되면 어떤 내부 절차를 따라야 하는가?
- 보호자 연락이 되지 않는 응급 상황에서 담당자는 무엇을 해야 하는가?
- 탈출, 교상, 상해가 발생하면 어떤 순서로 보고하고 기록하는가?
- 시설·인력·교육·영업자 준수사항의 공식 근거는 무엇인가?
- 고객 취소·환불·보상 문의에 어떤 문서가 적용되는가?
- 내부 매뉴얼이 최신 공식 규정과 충돌하거나 오래되었는가?
- 강아지가 초콜릿, 포도, 자일리톨 등 위험 물질을 먹었을 때 무엇을 해야 하는가?
- 사료 변경 뒤 설사하거나 구토할 때 어떤 순서로 관찰해야 하는가?
- 분리불안, 짖음, 배변 실수, 공격성은 어떻게 관리하고 언제 전문가 상담이 필요한가?
- 노령 반려동물의 물 섭취, 배뇨, 식욕 변화는 어떻게 기록하고 병원 상담으로 연결하는가?
- 고양이 화장실, 실내 놀이, 산책, 더위·추위 대응 등 생활 환경은 어떻게 정리하는가?

## 내부 합성 문서

초기 개발에는 실제 업체 문서 대신 다음 합성 문서를 사용합니다.

- 시설 기본 정보와 담당 체계
- 입실·퇴실 및 보호자 확인 SOP
- 위생·소독·분리 보호 SOP
- 탈출·상해·질병 의심 사고 대응 절차
- 예약 취소·환불·보상 내부 정책
- 건강 관찰과 병원 상담 기준
- 독성 물질과 응급 대응
- 식이·간식·체중 관리
- 행동 문제와 환경 풍부화
- 예방 관리와 노령 반려동물 케어

합성 문서는 법적 의무를 선언하지 않으며, 검색·인용·충돌 탐지 기능을 개발하기 위한 테스트 자료입니다.

## 첫 번째 수직 기능

1. 출처 카탈로그 API로 현재 사용할 문서와 수집 상태를 확인한다.
2. 합성 내부 문서를 파싱해 제목, 절, 버전, ACL을 보존한다.
3. 평가 질문 seed로 키워드 검색 기준선을 측정한다.
4. 문서 근거가 없을 때 AI 일반 조언 fallback을 별도 `answer_type`으로 표시한다.
5. 이후 임베딩/LLM/벡터 저장소를 ADR로 선정한다.

## 개발용 LLM 기준

- 기본 공급자: OpenAI Responses API
- 기본 모델: `gpt-4.1-mini`
- 출력 상한: 요청당 500토큰
- 입력 상한: 요청당 24,000문자
- 응답 저장: 비활성화
- 실제 API 호출 테스트: 명시적으로 `external` 테스트를 실행할 때만 허용

모델은 답변 작성에만 사용하고 문서 검색과 ACL 판정은 애플리케이션이 담당합니다. 기본은 문서 근거 우선이며, 사용자가 AI 일반 조언을 허용한 경우에만 문서 외 질문을 `openai_general` 모드로 처리합니다.

## 안전한 답변 정책

- 문서 근거 답변에는 문서명, 위치, URL 출처를 포함한다.
- 근거 문서와 위치를 찾지 못하고 AI 모드가 꺼져 있으면 답변하지 않는다.
- AI 일반 조언은 문서 기반 답변과 구분하고 안전 고지를 함께 표시한다.
- 의료 진단, 약물 용량, 응급 처치는 수의사 문의로 전환한다.
- 법적 책임이나 배상액을 단정하지 않는다.
- 내부 정책과 공식 규정이 다르면 둘을 분리해 보여주고 충돌 경고를 표시한다.
- 시행일 또는 문서 버전이 확인되지 않은 규정은 `최신성 미확인`으로 표시한다.
