# Security Policy

## 저장소에 넣지 말아야 할 것

- LLM/클라우드/DB API 키와 인증 토큰
- 실제 사내 문서, 개인정보, 고객 데이터, 계약서 원본
- 모델 입력/출력 전문이 포함된 운영 로그
- 접근 URL에 서명 토큰이나 자격 증명이 포함된 파일

유출이 의심되면 커밋 삭제보다 먼저 자격 증명을 폐기하고 재발급합니다. 이후 저장소 관리자에게 비공개 채널로 커밋 SHA와 노출 범위를 알립니다.

## 필수 위협 모델

- 악성 문서의 프롬프트 인젝션
- 문서 ACL 누락 또는 테넌트 간 정보 유출
- 업로드 파일의 악성 콘텐츠, 압축 폭탄, 위장 확장자
- 외부 URL 수집기의 SSRF 및 사설망 접근
- 로그/트레이스/캐시를 통한 데이터 잔존
- 과도한 요청 또는 긴 컨텍스트로 인한 비용 폭증
- 출처가 없는 환각 답변과 잘못된 고위험 조언

## 기본 통제

- 허용 확장자, MIME, 크기, 페이지 수를 서버에서 검증합니다.
- 외부 URL은 허용 도메인과 공인 IP만 수집하고 redirect도 재검증합니다.
- 원본과 파생 데이터에 동일한 테넌트/문서 ACL을 유지합니다.
- 검색 시점에 ACL을 적용하고 인용 직전에 다시 검증합니다.
- 공급자에 전송되는 데이터와 보존 정책을 문서화합니다.
- rate limit, timeout, 최대 토큰, 최대 검색 결과 수를 둡니다.
- 운영 환경은 디버그 모드와 상세 오류 응답을 끕니다.
- `OPENAI_API_KEY`는 `.env` 또는 배포 Secret에만 저장하고 코드, 문서, 스크린샷, 로그에 출력하지 않습니다.
- 모델 호출은 입력·출력 상한과 타임아웃을 적용하고, 검색 근거가 없으면 호출하지 않습니다.
- 실제 내부 문서를 외부 모델에 보내기 전 데이터 소유자 승인과 공급자 보존 정책을 확인합니다.

## 보안 이슈 신고

공개 Issue에 민감한 재현 데이터나 공격 절차를 올리지 않습니다. 팀 리더 또는 저장소 관리자에게 비공개로 공유하고, 조치 후 필요한 범위만 공개 기록으로 남깁니다.
